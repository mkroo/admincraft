# Editor 가이드

Editor는 AI 또는 코드로 어드민 페이지를 만들고, 갤러리에 공유하는 생산자입니다. 개발자에게 요청하지 않고 직접 페이지를 만들 수 있습니다.

## 해야 할 것

### 1. 페이지 만들기

AI 프롬프트 또는 TSX 코드로 페이지를 생성합니다.

| 방법 | 대상 | 설명 |
|------|------|------|
| [AI로 만들기](/guide/creating-pages-ai) | 비개발자, 빠른 프로토타이핑 | 자연어로 설명하면 AI가 코드 생성 |
| [코드로 만들기](/guide/creating-pages-code) | 개발자, 세밀한 제어 | Monaco 에디터에서 TSX 직접 작성 |

### 2. 페이지 테스트 & 수정

실제 배포 전에 API 환경을 전환하며 동작을 검증합니다. 생성된 페이지를 미리보기로 확인하고 수정합니다.

- **실시간 미리보기**: 코드 변경 즉시 우측 패널에 반영
- **API 환경 전환**: 에디터 상단 드롭다운으로 dev/staging/prod 전환
- **대화형 수정**: AI에게 "검색 필터 추가해줘" 같은 후속 요청 가능

### 3. 갤러리에 공유

갤러리에 공개해야 Viewer가 구독할 수 있습니다. 완성된 페이지를 갤러리에 공개하세요.

→ [페이지 갤러리](/guide/page-gallery)

::: tip 공개 전 확인
갤러리에 공개하면 모든 사용자가 구독할 수 있습니다. dev 환경이 아닌 prod 환경에서 정상 동작하는지 확인 후 공개하세요.
:::

## 할 수 있는 것

### AI 페이지 생성

자연어 프롬프트를 입력하면 AI가 등록된 OAS3 API 스펙을 기반으로 코드를 스트리밍 생성합니다. 테이블, 검색 폼, 대시보드 등 어드민에서 자주 쓰는 패턴을 AI가 알아서 구성합니다.

### Monaco 코드 에디터

VS Code와 동일한 엔진의 코드 에디터를 사용합니다. 자동완성, 구문 강조, 에러 표시를 지원합니다.

### API 환경 자유 전환

개발·스테이징·운영 서버를 드롭다운 하나로 전환할 수 있습니다. 개발 서버에서 테스트하고 운영 서버로 전환하여 실제 데이터를 확인하세요.

→ [API 환경 설정](/configuration/api-environments)

### 갤러리 공유 & 구독자 관리

페이지를 갤러리에 공개하고, 구독자 수를 확인하고, 공개/비공개를 전환할 수 있습니다.

### 개인 메뉴 구성

사이드바 메뉴를 자유롭게 구성할 수 있습니다. 직접 만든 페이지와 구독한 페이지를 폴더별로 정리하세요.

→ [메뉴 관리](/guide/menu-management)

::: info Editor가 할 수 없는 것
- 시스템 설정 변경 (AI provider, OAS3 스펙 등록) → Admin에게 요청
- 다른 사용자의 페이지 편집·삭제 → Admin 권한 필요
- API 환경(서버 URL) 추가·수정 → Admin이 OAS3 스펙에서 관리
:::

## 장점

### 개발 요청 없이 직접 만든다

JIRA 티켓을 만들고 개발자를 기다릴 필요가 없습니다. 필요한 어드민 페이지를 프롬프트 한 줄로 바로 생성할 수 있습니다.

### 코딩 몰라도 전문적인 페이지

AI가 설정된 디자인 시스템 컴포넌트를 사용하여 일관된 UI를 만들어줍니다. 여러 사람이 만들어도 통일된 화면이 됩니다.

### 대화로 수정하는 개발 경험

"검색 필터 추가해줘", "상태 컬럼을 뱃지로 바꿔줘" — 대화를 이어가며 페이지를 발전시킬 수 있습니다. 이전 대화 히스토리가 유지되어 맥락을 다시 설명할 필요가 없습니다.
